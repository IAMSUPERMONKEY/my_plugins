# HarmonyOS 日历权限申请方法

在 HarmonyOS 中使用 Calendar Kit（日历服务）需要申请相应的权限。以下是完整的申请流程：<rsup>1</rsup><rsup>2</rsup>

## 一、日历权限类型

| 权限名称                               | 说明                                        | 权限等级     | 授权方式                          |
| -------------------------------------- | ------------------------------------------- | ------------ | --------------------------------- |
| `ohos.permission.READ_CALENDAR`        | 读取日历信息                                | normal       | user_grant（用户授权）            |
| `ohos.permission.WRITE_CALENDAR`       | 添加、移除或更改日历活动 <rsup>3</rsup>     | normal       | user_grant（用户授权）            |
| `ohos.permission.READ_WHOLE_CALENDAR`  | 读取所有日历信息（包括其他应用创建的）      | system_basic | user_grant（受限权限，需申请ACL） |
| `ohos.permission.WRITE_WHOLE_CALENDAR` | 添加、移除或更改所有日历活动 <rsup>3</rsup> | system_basic | user_grant（受限权限，需申请ACL） |

> **说明**：大部分应用只需申请 `READ_CALENDAR` 和 `WRITE_CALENDAR` 即可满足需求。

## 二、申请步骤

### 步骤1：在 module.json5 中声明权限

在 `entry/src/main/module.json5` 文件中添加权限声明：

```json
{
  "module": {
    "name": "entry",
    "type": "entry",
    // ... 其他配置
    "requestPermissions": [
      {
        "name": "ohos.permission.READ_CALENDAR",
        "reason": "$string:read_calendar_reason",
        "usedScene": {
          "abilities": ["EntryAbility"],
          "when": "always"
        }
      },
      {
        "name": "ohos.permission.WRITE_CALENDAR",
        "reason": "$string:write_calendar_reason",
        "usedScene": {
          "abilities": ["EntryAbility"],
          "when": "always"
        }
      }
    ]
  }
}
```

### 步骤2：在资源文件中添加权限说明

在 `entry/src/main/resources/base/element/string.json` 中添加：

```json
{
  "string": [
    {
      "name": "read_calendar_reason",
      "value": "用于读取日历日程信息"
    },
    {
      "name": "write_calendar_reason",
      "value": "用于添加和管理日历日程"
    }
  ]
}
```

### 步骤3：在代码中动态申请权限

**推荐在 EntryAbility.ets 文件中申请权限**：<rsup>4</rsup>

```typescript
// entry/src/main/ets/entryability/EntryAbility.ets
import { abilityAccessCtrl, AbilityConstant, common, PermissionRequestResult, Permissions, UIAbility, Want } from '@kit.AbilityKit';
import { BusinessError } from '@kit.BasicServicesKit';
import { calendarManager } from '@kit.CalendarKit';
import { window } from '@kit.ArkUI';
import { hilog } from '@kit.PerformanceAnalysisKit';

const DOMAIN = 0x0000;

// 导出日历管理器供其他页面使用
export let calendarMgr: calendarManager.CalendarManager | null = null;
export let mContext: common.UIAbilityContext | null = null;

export default class EntryAbility extends UIAbility {
  onCreate(want: Want, launchParam: AbilityConstant.LaunchParam): void {
    hilog.info(DOMAIN, 'testTag', '%{public}s', 'Ability onCreate');
  }

  onDestroy(): void {
    hilog.info(DOMAIN, 'testTag', '%{public}s', 'Ability onDestroy');
  }

  onWindowStageCreate(windowStage: window.WindowStage): void {
    hilog.info(DOMAIN, 'testTag', '%{public}s', 'Ability onWindowStageCreate');

    windowStage.loadContent('pages/Index', (err, data) => {
      if (err.code) {
        hilog.error(DOMAIN, 'testTag', 'Failed to load the content. Cause: %{public}s', JSON.stringify(err));
        return;
      }
      hilog.info(DOMAIN, 'testTag', 'Succeeded in loading the content.');
    });

    // 获取上下文
    mContext = this.context;

    // 定义需要申请的权限
    const permissions: Permissions[] = [
      'ohos.permission.READ_CALENDAR',
      'ohos.permission.WRITE_CALENDAR'
    ];

    // 创建权限管理器
    let atManager = abilityAccessCtrl.createAtManager();

    // 向用户申请权限
    atManager.requestPermissionsFromUser(mContext, permissions)
      .then((result: PermissionRequestResult) => {
        hilog.info(DOMAIN, 'testTag', 'get Permission success, result: %{public}s', JSON.stringify(result));
        // 权限申请成功后，获取日历管理器
        calendarMgr = calendarManager.getCalendarManager(mContext);
      })
      .catch((error: BusinessError) => {
        hilog.error(DOMAIN, 'testTag', 'get Permission error, error. Code: %{public}d, message: %{public}s',
          error.code, error.message);
      });
  }

  onWindowStageDestroy(): void {
    hilog.info(DOMAIN, 'testTag', '%{public}s', 'Ability onWindowStageDestroy');
  }

  onForeground(): void {
    hilog.info(DOMAIN, 'testTag', '%{public}s', 'Ability onForeground');
  }

  onBackground(): void {
    hilog.info(DOMAIN, 'testTag', '%{public}s', 'Ability onBackground');
  }
}
```

## 三、权限申请工具类封装（可选）

为了方便在多个地方使用，可以封装一个权限管理工具类：

```typescript
// utils/PermissionManager.ets
import { abilityAccessCtrl, bundleManager, common, Permissions } from '@kit.AbilityKit';

export class PermissionManager {
  /**
   * 检查权限是否已授权
   */
  public static checkPermissions(permissionsArr: Permissions[]): boolean {
    const atManager = abilityAccessCtrl.createAtManager();
    const bundleInfo = bundleManager.getBundleInfoForSelfSync(
      bundleManager.BundleFlag.GET_BUNDLE_INFO_WITH_APPLICATION
    );
    const tokenID = bundleInfo.appInfo.accessTokenId;

    let result: boolean = true;
    permissionsArr.forEach((permission: Permissions) => {
      if (atManager.checkAccessTokenSync(tokenID, permission) !==
          abilityAccessCtrl.GrantStatus.PERMISSION_GRANTED) {
        result = false;
      }
    });
    return result;
  }

  /**
   * 申请权限
   */
  public static async requestPermissions(permissionList: Permissions[]): Promise<boolean> {
    const atManager = abilityAccessCtrl.createAtManager();
    const grantStatus = await atManager.requestPermissionsFromUser(
      getContext(),
      permissionList
    );
    const isAuth = grantStatus.authResults.every(
      v => v === abilityAccessCtrl.GrantStatus.PERMISSION_GRANTED
    );
    return isAuth ? Promise.resolve(true) : Promise.reject(false);
  }

  /**
   * 打开系统设置的权限管理页面
   */
  public static openPermissionSettingsPage() {
    const context = getContext() as common.UIAbilityContext;
    const bundleInfo = bundleManager.getBundleInfoForSelfSync(
      bundleManager.BundleFlag.GET_BUNDLE_INFO_WITH_APPLICATION
    );

    context.startAbility({
      bundleName: 'com.huawei.hmos.settings',
      abilityName: 'com.huawei.hmos.settings.MainAbility',
      uri: 'application_info_entry',
      parameters: {
        pushParams: bundleInfo.name
      }
    });
  }
}
```

## 四、使用示例

申请权限后，即可使用 Calendar Kit 进行日程管理：<rsup>5</rsup><rsup>2</rsup>

```typescript
// pages/Index.ets
import { calendarMgr } from '../entryability/EntryAbility';
import { calendarManager } from '@kit.CalendarKit';
import { BusinessError } from '@kit.BasicServicesKit';

@Entry
@Component
struct Index {
  build() {
    Column() {
      Button('添加日程')
        .onClick(() => {
          this.addCalendarEvent();
        })
    }
  }

  // 添加日程示例
  async addCalendarEvent() {
    try {
      // 获取默认日历账户
      const calendar = await calendarMgr?.getCalendar();

      // 构建日程对象
      const event: calendarManager.Event = {
        title: '测试日程',
        type: calendarManager.EventType.NORMAL,
        startTime: Date.now(),
        endTime: Date.now() + 60 * 60 * 1000, // 1小时后
        reminderTime: [10], // 提前10分钟提醒
        description: '这是一个测试日程'
      };

      // 添加日程
      const eventId = await calendar?.addEvent(event);
      console.info(`日程创建成功，ID: ${eventId}`);
    } catch (error) {
      const err = error as BusinessError;
      console.error(`添加日程失败: ${err.message}`);
    }
  }
}
```

## 五、注意事项

1. **权限申请时机**：建议在应用启动时就申请日历权限，推荐在 `EntryAbility.ets` 的 `onWindowStageCreate` 方法中申请。<rsup>4</rsup>

2. **用户拒绝授权**：如果用户拒绝授权，可以引导用户到系统设置中手动开启权限。<rsup>6</rsup>

3. **权限范围**：<rsup>7</rsup>
   - `READ_CALENDAR` 和 `WRITE_CALENDAR` 只能访问本应用创建的日程<rsup>8</rsup>
   - 如需访问其他应用的日程，需要申请 `READ_WHOLE_CALENDAR` 和 `WRITE_WHOLE_CALENDAR`（受限权限，需通过AGC申请ACL）

4. **隐私合规**：应用上架时需要在隐私政策中说明日历权限的使用目的。<rsup>3</rsup><rsup>9</rsup>

5. **系统设置**：系统设置页面的权限设置栏默认不展示，只有申请过权限后才会展示对应的权限开关。<rsup>7</rsup>
